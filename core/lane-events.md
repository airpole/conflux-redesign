# laneEvents — 레인 구분선 변형

> shape가 플레이필드 바깥 경계(Blue·Red)를 움직이듯, laneEvents는 안쪽 구분선 3개를 움직인다. 순수 시각 연출, 판정과 무관.
> 근거·결정 배경은 → [[rationale#laneEvents 설계 근거]]
> 태그: 전체 `[신규]`
> 구 `lineEvents`(`lines:[25,25,25,25]` 4칸 폭 단일 덩어리)는 편집 UI·렌더·게임 적용이 모두 미구현이고 실데이터도 균등 init 1개뿐이었다 → 보존·마이그레이션 대상 아님. laneEvents는 균등 기본(`lineNum 1/2/3` = `targetPos 0.25/0.5/0.75`)에서 출발하는 백지 설계. behavior-preserving의 예외이며 근거는 이 [신규] 태그 자체. (실측 → [[EXTRACTED_FACTS]] §9)

---

## 1. 개념

```
Blue │ 1 │ 2 │ 3 │ Red
 (L1)  (L2) (L3) (L4)
```

- **Blue / Red** = 바깥 경계. `shape`가 관할.
- **1 / 2 / 3** = 안쪽 구분선. `laneEvents`가 관할.
- 구분선이 움직이면 레인 폭이 시각적으로 변한다. 입력·판정에는 영향 없다 (노트는 자기 lane 키로 친다).
- init: 1·2·3은 Blue~Red를 균등 분할 (상대 0.25 / 0.5 / 0.75).

---

## 2. 데이터 모델

```
laneEvent = {
  startTick,
  duration,    // duration 공통 규칙: 0=step 점프, >0=easing 보간
  lineNum,     // ∈ {1,2,3} — 어느 구분선
  targetPos,   // 상대 좌표(실수, 경계 span 단위 — §3). 0=왼쪽 경계, 1=오른쪽 경계, 범위 제한 없음
  easing,      // shape와 동일: Linear / In-Sine / Out-Sine (null=anchor, 보간 안 함). Step·Arc는 입력 라벨 → [[shape]] §5
}
```

`chart.laneEvents` = 이 객체들의 배열.

shape 이벤트와 동형이다. 차이는 선택자 하나뿐: shape는 `isBlue`, laneEvents는 `lineNum`. 평가·캐시·편집 도구를 공유한다.

---

## 3. 좌표계 — 상대 단일 (전체비율)

`targetPos`는 **실수 전체**를 허용하는 상대 좌표다 `[번복 — 구 0~1 제한]`. 기준은 그 tick의 경계 span:

```
왼쪽 경계 = min(blueAt(tick), redAt(tick)),  오른쪽 경계 = max(blueAt(tick), redAt(tick))
데이터 위치 = lerp(왼쪽 경계, 오른쪽 경계, targetPos)   // 0=왼쪽, 1=오른쪽, 범위 제한 없음
```

- **0/1 기준은 동적** `[번복 — 구 "0=Blue, 1=Red" 고정]`: Blue·Red 중 그 tick에서 왼쪽에 있는 쪽이 0이다. 두 경계가 교차해도 lane 좌표는 항상 왼→오로 정규화되고, 교차 순간 span이 0을 지나므로 위치는 연속이다.
- 경계가 움직이면 구분선은 비율을 유지한 채 함께 스케일된다. **경계를 넘는 값(0 미만·1 초과)도 데이터로 유효하다** — 경계는 "레인이 보이는 창"이고, 구분선은 창 밖에 나가 있을 수 있다.

### 렌더 — 진실(shapes)과 투영(gameplay)
- **gameplay = 구속된 투영**: 구분선은 경계를 벗어나지 못하고, 순서(`Blue ≤ 1 ≤ 2 ≤ 3 ≤ Red`)를 지키도록 클램프되며, 경계·이웃과 **최소 간격**을 둔 채 따라다닌다(다닥다닥 붙되 분간은 되게 — 스케치 1→4 단계).
- **shapes 씬 = 진실**: 데이터 위치를 그대로 그리되, **gameplay 투영과 위치가 달라지는 구간은 점선**으로 그린다(경계 초과·순서 클램프 구간). 점선의 의미는 하나 — "지금 보이는 위치와 실제 게임 위치가 다르다".
- **`Blue == Red` (레인 폭 0)**: 모든 선이 한 점으로 수렴 — **하나의 선만** 그리고 구분선 1·2·3은 나타나지 않는다(최소 간격 규칙의 예외). 데이터는 그대로 유지되며 경계가 벌어지면 복원된다.

---

## 4. 구속 (Constraint)

**데이터에는 구속이 없다** `[번복 — 구 "순서 불변·역전 금지·드래그 롤백"]`. 순서 역전·경계 초과 모두 유효한 데이터이고, 에디터는 거부·롤백하지 않는다. 구속(`Blue ≤ 1 ≤ 2 ≤ 3 ≤ Red` + 경계 이내 + 최소 간격)은 **gameplay 렌더가 투영 단계에서** 지킨다(§3). symmetry·mirror의 대칭 결과가 범위 밖이어도 그대로 저장된다 — 위반 검사 자체가 없다. 근거 → [[rationale#lane을 무구속 데이터 + 투영 렌더로 뒤집은 이유]].

판정 코어는 laneEvents를 모른다. laneEvents를 아는 건 render와 editor뿐.

---

## 5. 편집 (Editor)

별도 탭이 아니다. 평소엔 shape 편집대로 동작하고, 필요할 때 구분선을 만져 laneEvent를 만든다. 같은 타임라인·드래그·easing을 공유하고 lineNum만 추가된다. easing은 shape와 100% 동일(저장 3종 + Arc 입력모드) → [[shape]] §5.

- **가로 스냅 그리드 = 전용 분할 수 `laneGridDivisor`** `[번복 — 구 gridDivisor 공유]`: 경계 span을 N등분 — 드롭다운 `2/3/4/6/8/12/16` + 정수 직접 입력, **기본 4**. Beatgrid 가로줄마다 점이 찍히고, 점 좌표는 `k/N`(k ∈ 정수)으로 **0·1 바깥까지 같은 간격으로 연장**된다(… −1/N, 0, 1/N, … 1, (N+1)/N …). 시간축 스냅은 여전히 노트의 gridDivisor([[timing]] §6) — 시간 분박과 공간 분할은 독립 축. (데이터는 상대 소수, 입력만 격자에 붙음) 근거 → [[rationale#lane 가로 그리드를 전용 분할 수로 분리한 이유]]
- **symmetry**: 대칭축은 **구분선 쌍(1-2 / 2-3 / 1-3) 중 선택**하고, **축 기본값 = 배치 지점에서 grid 스냅 기준 가장 가까운 tick 시점의 쌍 평균 위치**, 드래그로 조절. (`[번복]`: 이전 "쌍 init 중점"을 되돌림 — shape symmetry와 규칙 공유, [[shape]] §6과 동일한 동적 스냅샷.) 배치 클릭은 쌍의 오른쪽 구분선 위치를 찍고 왼쪽이 축 대칭으로 자동 생성된다(입력 UI·쌍 지정 키는 [[editor-editing]] §3): `targetPos′ = 2×축 − targetPos`, lineNum은 쌍의 반대편. 한 입력으로 대칭 다중 이벤트를 만들지만, 데이터 단위는 여전히 "구분선 하나당 이벤트 하나".
- **그룹 조작** `[신규]`: `Q`/`W`/`E`(line1/2/3)를 **조합으로 유지**하면(QW/WE/QE/QWE) 그 라인들이 그룹이 된다. 그룹 상태에서 `R` = **간격 유지 이동 ↔ pinch 전환** 스위치. 간격 유지 이동은 **가장 오른쪽 라인이 커서를 따르고** 나머지가 오프셋을 유지하며 따라온다; pinch는 그룹 전체를 클릭 위치 한 점에 배치한다. 같은 키 조합이 S ON이면 symmetry 쌍 지정(위 항목), S OFF면 그룹 조작 — 모드로 의미를 가른다. 키 표는 [[editor-editing]] §2.
- **init 이동**: shape의 init점을 옮길 수 있는 것처럼, 구분선의 init 위치(기본 균등 0.25/0.5/0.75)도 차트별로 옮길 수 있다.

---

## 6. 평가 (Core)

**chain 평가 알고리즘은 [[shape]] §4가 단일 출처다.** laneEvents는 같은 알고리즘을 쓰고, 다른 점은 둘뿐이다:

- **선택자**: shape는 `isBlue`로 Blue·Red 두 체인, lane은 `lineNum`으로 구분선 1·2·3 세 체인을 가른다.
- **반환·좌표계**: `laneLayoutAt(tick)` → `{ line1, line2, line3 }` (각 상대 실수 — 무구속, §3). px 변환·경계/순서 클램프·최소 간격은 render의 투영 단계 몫.

나머지(anchor=`easing===null`로 값 못박기 / 보간 이벤트 정렬·순회 / `duration===0` 점프 / easing 보간)는 shape §4와 동일하다. anchor가 없으면 init 균등(0.25/0.5/0.75) fallback.
